You are part of a system building a web application called **The Bridge**.

## 🌉 Product Concept

The Bridge is an astrology experience that integrates:

* Tropical astrology → “The Earth Self” (lived experience, personality, conditioning)
* Sidereal astrology → “The Star Self” (archetypal force, deeper trajectory)

The purpose is not to generate generic astrology readings.

The purpose is to:

1. Translate chart data into symbolic traits
2. Detect patterns and tensions between two systems
3. Synthesize those into “Bridge Themes”
4. Generate a structured, poetic reading

---

## 🧠 System Philosophy

* A person is not one identity, but a layered experience
* Tropical = how life is experienced
* Sidereal = what is moving through the person
* The Bridge = where these meet

We do NOT treat astrology as fixed definitions.

We treat it as:
→ symbolic pattern recognition

---

## 🏗️ System Architecture

INPUT (birth data)
→ Chart Engine (planetary positions)
→ Trait Engine (symbolic traits)
→ Bridge Engine (pattern detection)
→ Render Engine (reading generation)
→ UI (user experience)

Each layer must be:

* modular
* deterministic (except optional language polishing)
* composable

---

## 🎨 Tone Guidelines

All writing must be:

* warm, grounded, and poetic
* emotionally intelligent
* non-preachy
* not overly verbose
* not repetitive
* not absolute (“you are…”)
* not generic astrology language

The tone should feel like:
→ a reflection, not a diagnosis

---

## 🚫 Avoid

* Hard-coded full readings
* Random generation without structure
* Mixing logic and writing layers
* Overengineering

---

## ✅ Goal

Build a working MVP that:

* generates tropical + sidereal placements
* extracts symbolic traits
* detects 10–20 bridge themes
* renders a structured reading

Focus on:
→ clarity
→ modularity
→ speed to working prototype
